Milestone 005: Evidence Memory Foundation

Give the decision cycle bounded, inspectable continuity across frames. Memory should retain attributed observation evidence without claiming a complete world model, changing perception semantics, or granting movement authority before its lifecycle and value are understood.

Status: active - implementation Proposed: 2026-07-15 Action policy: idle Persistence: memory only by default Hosts: Chase and DonkeyCar Visual gates: Chase and Pi Delivery: rolling PR horizon

High-Level Objective

Retain Evidence, Not Truth

Carry selected observations across frames with source, time, confidence, and uncertainty intact. Memory records what was observed and retained, not what definitively exists.

One Lifecycle Across Hosts

Run the same memory contract and implementation selection in the local Chase controller and onboard Pi host without simulator map input or vehicle-specific memory types.

Bounded And Inspectable

Make capacity, expiry, reset, provenance, and current state visible to operators. Default execution remains in memory and writes no frame or history artifacts.

Success is not a detailed floor plan. Success is a stable, economical answer to "what recent evidence is the controller retaining, why, and for how long?" that behaves the same in replay, simulation, and the physical runtime while the car remains idle. In both Chase and on the Pi, that answer must be visually reviewable against the exact source frames that produced the retained evidence; schema compatibility and record counts alone are not enough to close the milestone.

Completion Usage

These proposed commands define how milestone 005 should become runnable; none are implemented yet. Names, options, schemas, and presentation may sharpen during delivery, but removing or adding a workflow requires an explicit scope decision.

Select And Verify Memory

Starting state: a vehicle appears in vehicles active and a packaged memory implementation is available.

Proposed execution:

./cli/automa vehicles active
./cli/automa vehicles update memory \
  --id <vehicle-id> \
  --implementation bounded_evidence
./cli/automa vehicles info memory --id <vehicle-id>

Success signal: concise output names the selected implementation, release, capacity, expiry policy, and activation health. --json exposes the same selection for automation.

Run And Inspect Retained Evidence

Starting state: memory and the idle decision engine are selected for the vehicle.

Proposed execution:

./cli/automa vehicles automation run --id <vehicle-id>
./cli/automa vehicles stream memory --id <vehicle-id>

Success signal: a replacing live view reports lifecycle health, retained record count, age, capacity, provenance summaries, and explicit unavailable or failure state without writing logs by default.

Replay, Compare, And Reset

Starting state: a recorded observation sequence and selected memory configuration are available.

Proposed execution:

./cli/automa vehicles memory replay \
  <observation-run> --id <vehicle-id>
./cli/automa vehicles memory reset --id <vehicle-id>
./cli/automa vehicles info memory --id <vehicle-id>

Success signal: replay reports a deterministic result digest and concise final state; reset reports an empty new lifecycle epoch. --json provides records and digests for automated comparison.

Verify Simulator Evidence Retention

Starting state: the Chase frontend is available and the bounded memory implementation is packaged for the local controller.

Proposed execution:

./cli/automa simulators ensure \
  --scenario chaser-depth-obstacles
./cli/automa vehicles update memory \
  --id chase-sim-chaser \
  --implementation bounded_evidence
./cli/automa vehicles memory check \
  --id chase-sim-chaser --record

Success signal: Chase's built-in decision model remains the movement authority while the rewritten cycle runs in shadow/observe-only mode. A live camera-derived run visibly links current perception to retained, expired, and reset memory records through exact source frames. A frame-aligned evaluator may compare the rewritten cycle with Chase debug/reference state, but that state never enters the controller or memory input. Closeout promotes a compact visual extract into tracked milestone evidence.

Verify Physical Evidence Retention

Starting state: the Pi publishes healthy frame-matched perception, the bounded memory implementation is active, and the idle engine owns action.

Proposed execution:

./cli/automa vehicles memory check \
  --id <picar-id> --record
./cli/automa vehicles info memory --id <picar-id>

Success signal: a stationary guided sequence records an exact perception source frame, its retained record, bounded survival through declared dropout, and visible expiry/reset. The report renders source-frame regions beside memory age and provenance; it never draws stale image coordinates on a newer frame as if they were current. Closeout promotes a compact visual extract into tracked milestone evidence.

Default runs remain ephemeral and concise. A user must opt in before a replay input, frame, log, or memory history is written to disk.

Stage Meaning

SensorSnapshot
  -> PerceptionText       current plugin evidence
  -> Observation          decision-facing representation of one moment
  -> MemorySnapshot       bounded evidence retained across moments
  -> Patterns             recurring relationships inferred from memory
  -> Projections          hypotheses about possible future state
  -> Action               immediate control output (idle throughout this milestone)

Observation

Answers what the current cycle made available. It is frame-scoped, detached for inspection, and may explicitly report missing or degraded perception.

Memory

Answers what attributed evidence remains relevant after one cycle. It owns retention and expiry, not semantic classification or future inference.

Patterns

Answer what relationships recur across retained evidence. They are later derived claims and are not required to make memory useful.

Projections And Action

Answer what may happen and what should be done now. They remain no-op so memory can be evaluated without movement masking its failures.

Memory Contract Constraints

ConcernRequired DirectionNot Yet Assumed
ProvenanceEvery retained claim cites source observation, plugin/evidence identity, coordinate frame, observed time, and update time.That two similar records are the same physical object.
ConfidenceRetain source confidence and make any aggregate confidence rule attributable.That repetition alone turns uncertain evidence into truth.
BoundsCapacity, age, and eviction behavior are finite, configured, visible, and testable.Unbounded history, raw frame archives, or an implicit database.
LifecycleConstruction, update, read, reset, activation replacement, and failure states have one owner.Persistence across process or device restart.
CoordinatesPreserve declared image or relative frames and reject incompatible fusion unless an implementation explicitly transforms them.Calibrated metric geometry or a shared global map.
InspectionReturn stable summaries and structured records detached from live mutable state. Retained image-space evidence preserves the exact source frame/observation identity needed for opt-in visual review.Dumping every internal index or implementation cache to operators, retaining raw frames by default, or projecting stale coordinates onto a newer frame without an explicit transform.
FailureA memory failure is explicit, produces no new retained claim, and cannot produce movement.Silently falling back to stale state as if it were current.

Complexity belongs in the stable lifecycle and safety boundary when it makes implementations smaller. A memory plugin should primarily express what to retain, update, merge, or expire; it should not duplicate activation parsing, status rendering, persistence policy, or host wiring.

Evaluation Contract

Deterministic Sequences

Use synthetic observations to prove recurrence, dropout, conflict, expiry, capacity, reset, failure isolation, and repeatable replay without CV variability.

Simulator Evidence

Let Chase's built-in model drive chaser-depth-obstacles while the rewritten cycle runs observe-only in shadow. Record a source-frame-to-memory visual review and a frame-aligned reference snapshot. Hidden scene state may score an implementation but never becomes its memory input.

Physical Evidence

Run a stationary present/dropout/expiry/reset sequence on the Pi. Record an opt-in review that pairs each retained item with the exact perception source frame and region while the idle engine retains zero steering and throttle.

Scope Boundaries

In Scope

  • Stable memory values, lifecycle, activation, reset, and failure behavior.
  • Simple loadable memory implementations with bounded state.
  • Equivalent local simulator and onboard Pi host integration.
  • Operator inspection, streaming, replay, and opt-in recording.
  • Deterministic sequence tests and live, visually reviewable simulator evaluation.
  • Non-moving physical activation, state inspection, and recorded visual-provenance validation.
  • Honest physical deploy verification that a shipped memory activation is actually live (or fails with core-update guidance).

Out Of Scope

  • Non-idle decision or action policy.
  • Path planning, navigation, collision avoidance, or map-based control.
  • Metric SLAM, calibrated global coordinates, or a complete floor plan.
  • Semantic object recognition or universal object identity.
  • Durable database, cloud memory, or default frame archives.
  • Model training, VLM integration, or privileged simulator state as input.
  • Merging core and autonomy into one deploy command, or rewriting manage.py ownership for this milestone.

Exit Criteria

  1. Stable memory input and output types replace public Any at the decision-cycle boundary without encoding a concrete mapping algorithm.
  2. Observation, memory, patterns, projections, and action have distinct documented meanings and stage ownership.
  3. Memory state has finite capacity and age, deterministic serialization, explicit reset, attributable provenance, and isolated failure behavior.
  4. At least one simple implementation retains and expires structured evidence without claiming semantic or metric world truth.
  5. The same activation and lifecycle run through local Chase and onboard Donkey hosts; neither receives privileged simulator map state.
  6. Automa can stage, inspect, run, stream latest memory, replay a sequence, and reset state with concise human output and complete machine output.
  7. Default execution writes no logs, frames, or memory history; recording is explicit and bounded.
  8. Deterministic tests cover recurrence, dropout, conflict, expiry, capacity, reset, failure, and replay, including long-sequence boundedness.
  9. One live chaser-depth-obstacles shadow check and one recorded non-moving Pi present/dropout/expiry/reset check verify equivalent stage behavior while the rewritten engine emits zero movement.
  10. During the simulator check, Chase's built-in decision model retains movement authority, the rewrite runs observe-only, and candidate/reference results are aligned by simulator frame identity.
  11. Simulator debug, map, and reference-decision state are evaluator-only and are absent from rewritten controller inputs and retained memory provenance.
  12. Tracked simulator and physical reviews visibly trace retained image-space evidence to exact perception source frames and regions, distinguish current from retained/expired evidence, and never treat stale coordinates as current geometry.
  13. A closeout states what memory representation proved useful, what remains unverified, and whether later pattern or action work is justified.
