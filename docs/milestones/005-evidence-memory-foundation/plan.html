Scaling And Design Risks

Plugin Identity

Ledger keys omit source_plugin_id. Two plugins using the same local thing or signal ID can silently overwrite one another. Namespace IDs at the framework boundary.

Checker Coupling

The generic CLI checker assumes BoundedEvidenceLedger policy: retained thing: keys, dropout survival, and fixed age expiry. Split host sampling from implementation-specific lifecycle assertions.

CLI Concentration

Transport, lifecycle evaluation, artifact writing, rendering, and human output are concentrated in large CLI modules. Extract cohesive adapters/evaluators only while repairing these paths; avoid a broad speculative framework.

Validation Blindspot

The major risk is testing synthesized checker behavior rather than external reality. A passing check must establish where each observation came from and which runtime actually changed.

High-Level Objective

Retain Evidence, Not Truth

Carry selected observations across frames with source, time, confidence, and uncertainty intact. Memory records what was observed and retained, not what definitively exists.

One Lifecycle Across Hosts

Run the same memory contract and implementation selection in the local Chase controller and onboard Pi host without simulator map input or vehicle-specific memory types.

Bounded And Inspectable

Make capacity, expiry, reset, provenance, and current state visible to operators. Default execution remains in memory and writes no frame or history artifacts.

Success is not a detailed floor plan. Success is a stable, economical answer to "what recent evidence is the controller retaining, why, and for how long?" that behaves the same in replay, simulation, and the physical runtime while the car remains idle. In both Chase and on the Pi, that answer must be visually reviewable against the exact source frames that produced the retained evidence; schema compatibility and record counts alone are not enough to close the milestone.

Stage Meaning

SensorSnapshot
  -> PerceptionText       current plugin evidence
  -> Observation          decision-facing representation of one moment
  -> MemorySnapshot       bounded evidence retained across moments
  -> Patterns             recurring relationships inferred from memory
  -> Projections          hypotheses about possible future state
  -> Action               immediate control output (idle throughout this milestone)

Observation

Answers what the current cycle made available. It is frame-scoped, detached for inspection, and may explicitly report missing or degraded perception.

Memory

Answers what attributed evidence remains relevant after one cycle. It owns retention and expiry, not semantic classification or future inference.

Patterns

Answer what relationships recur across retained evidence. They are later derived claims and are not required to make memory useful.

Projections And Action

Answer what may happen and what should be done now. They remain no-op so memory can be evaluated without movement masking its failures.

Risks And Controls

RiskControl
Memory silently becomes asserted world truth.Require provenance, source confidence, coordinate frame, recency, and explicit implementation claim boundaries on every retained record.
Plugin implementations inherit framework complexity.Put activation, bounds enforcement, lifecycle, timing, status, and failure isolation in stable infrastructure; keep plugins focused on update policy.
A generic memory framework is designed before one useful reducer exists.Land the smallest stable contract first, then one bounded implementation; add abstractions only from demonstrated duplication.
Hidden state grows with run length or survives reset.Test capacity, expiry, long sequences, activation replacement, reset, and serialized size directly.
Raw images become accidental memory or disk history.Store evidence references and compact attributes by default; require explicit recording for artifacts and define cleanup.
Simulator convenience leaks map truth into runtime input.Use only vehicle sensor-derived observations for memory. Hidden scene state is evaluator-only.
Memory failures influence movement.Keep action idle, isolate stage failure, and assert zero control in local and physical validation.
Operator output becomes an unbounded record dump.Default to health, counts, age, capacity, and concise summaries; provide complete structured detail only through machine output or explicit inspection.
