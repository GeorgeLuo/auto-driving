Donkey Server Functionality

This diagram explains the PiRacer DonkeyCar server we have been using on piracer.local:8887. The server is a Tornado web controller that stores manual-control state and owns the loadable autonomy manager. The Donkey vehicle loop runs at 20 Hz; its drive mode chooses whether manual controls or autonomy pilot controls reach the drivetrain.

Source points: deploy/targets/donkeycar/vendor/donkeycar/donkeycar/parts/web_controller/web.py, deploy/targets/donkeycar/app/manage.py, and implementations/runtime/donkeycar/donkey_part.py

Runtime Flow

Clients

Browser UI, automa, or other local clients issue HTTP/WebSocket commands.

  • GET /drive opens the UI.
  • POST /drive updates command state.
  • GET /frame.jpg reads the latest frame.
  • /autonomy/* inspects or switches autonomy.

Tornado LocalWebController

Owns the web routes, mutable manual state, and AutonomyManager.

  • angle, throttle
  • mode, recording
  • buttons, img_arr
  • selected engine, status, and last control

Vehicle Loop

V.start(rate_hz=20) calls each part in order.

  • run_threaded() publishes user controls.
  • AutonomyPilotPart runs only when run_pilot is true.
  • The shared cycle receives the current camera and user state.

Final Motion

DriveMode chooses user or pilot controls before drivetrain parts write PWM.

  • user uses browser/API/joystick values.
  • local uses autonomy pilot values.
  • PWMSteering writes steering pulse.
  • PWMThrottle_WS writes throttle pulse.

Camera Feed

The active camera publishes cam/image_array. The web controller keeps the latest array and serves it as JPEG or MJPEG.

Recording Path

When recording is true, the tub writer records camera frames and control values for training or later analysis.

Autonomy Activation

At startup, runtime/decision/active.json selects the engine. The current deployed engine is idle; the validated mode remains manual user.

Joystick Override

If started with --js, the joystick controller is added after the web controller and writes the same user control names.

Endpoint Map

Endpoint Purpose Notes
GET / Redirects to /drive. Convenience entry point.
GET /drive Serves the Donkey driving UI. The UI talks back to the same controller state.
POST /drive Sets angle, throttle, drive_mode, recording, and optional button latches. angle is the steering field consumed by this handler. A separate steering field is ignored by this endpoint.
WS /wsDrive Live bidirectional driving channel for the browser UI. Updates the same controller state as POST /drive.
GET /frame.jpg Returns the latest camera frame as a single JPEG. Returns 503 if the camera has not produced a frame yet.
GET /video Returns a multipart JPEG stream. Useful for browser preview, less useful for precise frame-by-frame analysis.
GET /frame-highres.jpg Captures and returns the highest-resolution still supported by the camera object. Runs capture work in an executor and includes width/height headers.
GET /autonomy/status Returns Donkey drive mode, mode mapping, selected engine, counters, errors, and last control. This is the post-deploy verification endpoint used by automa.
POST /autonomy/mode Switches between manual and autonomy. manual maps to Donkey user; autonomy maps to local. Returning to manual stops by default.
POST /autonomy/engine Loads an engine class and config into the current process. This is an in-memory administrative change. It does not rewrite the persisted activation manifest.
POST /autonomy/reload Reloads the currently selected engine module and config. Errors are returned as JSON while the manager retains inspectable error state.
GET /calibrate, WS /wsCalibrate Servo and throttle calibration UI/channel. Can update calibration pulses while the server is running.

Autonomy Startup

The web controller creates an AutonomyManager with the safe idle engine.

manage.py reads runtime/decision/active.json and applies the selected engine configuration.

AutonomyPilotPart is added to the Donkey loop under the run_pilot condition.

The server starts in manual user mode, so loading an engine alone does not grant movement authority.

Autonomy Tick

POST /autonomy/mode with {"mode":"autonomy"} maps Donkey mode to local.

The part wraps cam/image_array as a front-camera SensorSnapshot and runs the shared cycle.

The engine returns normalized steering/throttle as pilot/angle and pilot/throttle.

DriveMode applies those pilot values only while the mode remains local.

Control Sequence

Client posts a JSON command to /drive.

DriveAPI.post() writes values into LocalWebController.

At the next loop tick, run_threaded() returns user/steering, user/throttle, user/mode, recording, and web/buttons.

DriveMode selects manual or pilot controls. In user mode, the posted command is the final command.

Drivetrain parts convert normalized steering/throttle into PWM pulses and send them to the steering servo and motor controller.

Command Shape

For API-only control, send short pulses and then explicitly stop. The server holds the last command until another command changes it.

POST http://piracer.local:8887/drive
Content-Type: application/json

{
  "angle": 0.20,
  "throttle": 0.45,
  "drive_mode": "user",
  "recording": false
}
POST http://piracer.local:8887/drive
Content-Type: application/json

{
  "angle": 0.0,
  "throttle": 0.0,
  "drive_mode": "user",
  "recording": false
}

Move And Capture Pattern

Pulse Command

Post a steering/throttle command for a fixed duration.

Immediate Stop

Post zero steering and zero throttle so the held state becomes safe.

Read Current Frame

Fetch /frame.jpg after motion settles, or use /frame-highres.jpg for still analysis.

Analyze Offline

Store the frame, command metadata, and timing so local CV/VLM analysis can reconstruct what happened.

Important Override Behavior

The web controller is always added as a user controller, but joystick support can also be added. In this setup, the joystick controller writes user/steering, user/throttle, user/mode, and recording after the web controller. That means a connected controller can overwrite commands posted by scripts or the browser UI.

  • API/web control is now the default restart mode.
  • Use ./cli/automa vehicles update core --id piracer --restart --drive-args=--js when you want the handheld controller to be active.
  • Changing an engine does not change the drive mode; movement authority requires an explicit autonomy-mode request.
  • For autonomous experiments, choose one command source at a time and record which one is active.
