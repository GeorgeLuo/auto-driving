Onboard Autonomy Flow

The automation engine treats each vehicle as a sensor-and-control boundary. A shared, no-op-friendly cycle can perceive, observe, retain state, and select an action without requiring every stage to be implemented. The only current decision engine is deliberately idle, so the framework can be validated without autonomous movement.

Source points: autonomy/decision/cycle.py, autonomy/runtime/cycle_host.py, cli/automa_cli/automation.py, and implementations/runtime/donkeycar/donkey_part.py

Main Runtime Loop

Sensor Input

Each tick starts with a generic SensorSnapshot.

  • The current shared sensor is front_camera.
  • Chase captures it through the WS/CLI adapter.
  • DonkeyCar wraps cam/image_array in the same shape.

Optional Perception

A configured mapper turns sensor readings into PerceptionText evidence.

  • The stable stage is agnostic of sensor and evidence kinds.
  • Plugins declare and resolve their own input components.
  • Plugins emit signals, things, artifacts, and limits.

Observation And State

Perception is adapted to the stable decision-facing Observation shape.

  • The default observe stage runs only when perception exists.
  • Memory, patterns, and projections are optional stages.
  • All three optional state stages are currently no-ops.

Loadable Engine

AutonomyCycleHost gives the cycle result to the engine selected by AutonomyManager.

  • Selection comes from runtime/decision/active.json.
  • The current idle engine always returns zero control.
  • Engines receive evidence, never privileged simulator map state.

Runtime Control Gate

The normalized AutonomyControl is applied by the environment-specific host.

  • Chase currently enforces a stop-only safety gate.
  • DonkeyCar publishes pilot outputs only in pilot mode.
  • Manual user mode remains outside autonomy control.

On PiRacer

AutonomyPilotPart adapts Donkey vehicle-memory values to the shared cycle at a bounded cadence in manual mode. It retains one detached latest frame/result snapshot and publishes it on /autonomy/observation/latest (plus matching frame.jpg). Perception and decision activations select the plugins and engine; pilot outputs stay zero while drive mode remains user. Automa inspects the path through stream, local overlay, guided check, and viability measurement without hosting a second Pi frontend.

In Simulation

The local worker captures only the Chase front view, runs staged perception, and records the latest cycle. It takes WS control but commands stop on every current tick.

Persistence

Continuous simulation overwrites latest frame/state files by default. --record opts into timestamped artifacts; no worker log is written without --log.

Current Implemented Behavior

Capture

The runtime creates a DecisionFrameContext containing the current SensorSnapshot and frame metadata.

Perceive

The activated generic plugin-chain stage passes a component context over the sensor snapshot. Each configured implementation resolves only the components it needs. The same stage shape runs locally for Chase and on the Donkey host.

Observe

When perception exists, the cycle automatically creates a decision_observation_v0 record. With no perception, observation is also null.

Leave State Stages Empty

No live memory, pattern, or projection implementation is selected yet. The cycle explicitly carries null for those outputs.

Run Idle Engine

The selected engine returns AutonomyControl(steering=0, throttle=0) with an inspectable reason and confidence.

Publish Latest Snapshot

On DonkeyCar, the part detaches the exact processed frame and findings into one in-memory latest snapshot. Read-only HTTP endpoints expose JSON and JPEG with matching frame identity and read-time health. Default operation writes no perception history.

Apply Environment Gate

Chase remains stopped. On DonkeyCar, pilot values affect the drivetrain only after an explicit switch from manual to autonomy mode. Observation itself does not require pilot mode.

Memory remains a contract, not an implementation: the shared cycle carries explicit memory, pattern, and projection slots, but all three remain null. Physical observation is proven under manual mode; a memory implementation should consume real Observation values and appear directly in DecisionCycleResult rather than manufacturing a parallel inspection-only representation.

Layer Contracts

Layer Input Output Current Code
Vehicle Read request or bounded operation command. SensorSnapshot, pulse result, and stop. autonomy/vehicle/vehicle.py
Perception Sensor-agnostic PerceptionRequest; plugins query components from its snapshot. PerceptionText containing evidence and limits. autonomy/perception/, implementations/perception/
Observation Current snapshot and optional perception. Stable decision-facing evidence record. autonomy/decision/observation.py
State Stages Observation and prior stage output. Optional memory, patterns, and projections; currently null. autonomy/decision/cycle.py
Decision AutonomySnapshot with all available stage outputs. Normalized AutonomyControl. autonomy/runtime/engine.py, cycle_host.py
Operations CarInterface plus an explicitly invoked plan. Bounded action results and optional reports. implementations/operations/

Inspection Commands

Live Controller

Inspect staged contracts and the latest local worker result.

./cli/automa vehicles info perception --id chase-sim-chaser

./cli/automa vehicles info decision --id chase-sim-chaser

./cli/automa vehicles stream perception --id chase-sim-chaser

Recorded Cycle

Opt into a bounded recording when the full perception, observation, control, and cycle result need inspection.

./cli/automa vehicles automation run --id chase-sim-chaser --frames 5 --record

Recorded cycle JSON reflects the actual runtime; unimplemented state stages remain null.
